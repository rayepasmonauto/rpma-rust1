# 05. IPC API AND CONTRACTS

The IPC layer connects the React frontend and Rust backend through Tauri command handlers.

## Contract Rules

- Rust handlers should return `ApiResponse<T>` at the boundary.
- Frontend code should call `safeInvoke()` or a domain IPC wrapper, never raw `invoke()`.
- Protected commands receive the current session automatically through `safeInvoke()`.
- `correlation_id` should flow through the request path for tracing.
- Authentication and RBAC belong in Rust request-context resolution, not in the frontend.

## Response Envelope

`src-tauri/src/shared/ipc/response.rs`

```rust
pub struct ApiResponse<T> {
    pub success: bool,
    pub message: Option<String>,
    pub error_code: Option<String>,
    pub data: Option<T>,
    pub error: Option<ApiError>,
    pub correlation_id: Option<String>,
}
```

## Type Sync

Rust types marked with `#[derive(TS)]` and `#[ts(export)]` are exported to TypeScript.

| Step | Command | Output |
|---|---|---|
| Generate | `npm run types:sync` | `frontend/src/lib/backend.ts` |
| Drift check | `npm run types:drift-check` | Detects contract mismatch |

`frontend/src/lib/backend.ts` is generated by `scripts/write-types.js` and must not be edited by hand.

## Command Registration

Handlers are registered in `src-tauri/src/main.rs` with `tauri::generate_handler![...]`.

## Adding A New IPC Command

1. Define or update request and response types.
2. Implement the Rust handler in the correct domain IPC module.
3. Register the handler in `src-tauri/src/main.rs`.
4. Add the frontend wrapper under `frontend/src/domains/<domain>/ipc/*`.
5. Add or reuse query keys in `frontend/src/lib/query-keys.ts` when server state is involved.
6. Run `npm run types:sync` if any exported Rust type changed.
